Phase 1 — Feedback Management System

1. Project Title

Feedback Management System

2. Project Overview

Organizations, training institutes, educational platforms, and enterprises regularly collect feedback from participants, employees, customers, or trainees to evaluate the quality of services, training programs, products, and overall experience.

Currently, many organizations collect feedback using:

· Google Forms

· Excel sheets

· Emails

· Manual surveys

These approaches often result in:

· Scattered feedback data

· Difficult reporting and tracking

· No centralized management

· Limited search capability

· No structured storage for future analytics

To improve the feedback collection and management process, the organization wants to build a centralized web-based Feedback Management System using:

· Frontend: React

· Backend: Python FastAPI

· Database: SQLite/PostgreSQL

This Phase 1 implementation focuses on developing the foundational full-stack application with core feedback management capabilities. The system should also be designed to support future extensions involving:

· Data Engineering pipelines

· Feedback analytics dashboards

· Sentiment analysis

· AI-powered semantic search

· GenAI-based summarization systems

3. Business Problem Statement

The organization currently lacks a centralized mechanism for managing and analyzing feedback data.

Current challenges include:

· Feedback stored across multiple platforms

· Difficulty identifying common issues

· Manual consolidation of responses

· No efficient search capability

· Limited historical analysis

· Poor visibility into participant satisfaction trends

The proposed application will centralize feedback collection, storage, search, and management into a single platform.

4. Project Objective

The objective of this capstone project is to design and develop a scalable Feedback Management System that allows users to:

· Submit feedback

· View submitted feedback

· Search feedback records

· Filter feedback based on ratings or categories

· Manage feedback data centrally

The application should follow industry-standard full-stack development practices and REST API design principles.

5. Scope of Phase 1

Phase 1 includes:

· Full-stack application development

· REST API implementation

· Feedback CRUD operations

· Search and filtering functionality

· Database integration

· Frontend-backend integration

Phase 1 does NOT include:

· Authentication and authorization

· AI/ML integration

· Sentiment analysis

· Dashboard analytics

· Semantic search

· Cloud deployment

6. Functional Requirements

6.1 Submit Feedback

The system should allow users to submit feedback by providing:

· Participant Name

· Training/Event/Product Name

· Rating

· Feedback Comments

6.2 View Feedback

The system should allow users to:

· View all feedback records

· View detailed feedback entries

· View ratings and comments

· View submission timestamps

6.3 Update Feedback

Administrators should be able to:

· Modify feedback details

· Update ratings

· Edit comments if necessary

6.4 Delete Feedback

The system should allow deletion of feedback entries when required.

6.5 Search and Filter Feedback

The application should support:

· Keyword-based search

· Rating-based filtering

· Training/Event-based filtering

7. Rating Levels

The system should support ratings between:

· 1 to 5

Example:

· 1 → Poor

· 2 → Fair

· 3 → Good

· 4 → Very Good

· 5 → Excellent

User Type Responsibilities

10. API Requirements

Method Endpoint Description

GET /feedback Retrieve all feedback

GET /feedback/{id} Retrieve feedback by ID

POST /feedback Submit feedback

PUT /feedback/{id} Update feedback

DELETE /feedback/{id} Delete feedback

GET /search Search feedback
